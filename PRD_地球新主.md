## 1. 项目概述

### 1.1 产品简介

《地球新主》是一款基于LBS（Location-Based Service）的iOS策略生存游戏。游戏设定在末日后的世界，玩家通过真实世界的行走来占领土地、探索资源、建造设施，与其他幸存者交易和通讯，逐步成为这片土地的新主人。

### 1.2 产品愿景

让玩家在日常行走中体验末日生存的乐趣，将现实世界变成游戏战场，鼓励户外活动的同时提供沉浸式的策略游戏体验。

### 1.3 核心卖点

- **真实行走圈地**：GPS追踪玩家路径，闭环即占领
- **现实POI探索**：医院、超市等真实地点成为资源采集点
- **社交生存**：与其他玩家交易、通讯、竞争领地

### 1.4 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | SwiftUI |
| 地图服务 | MapKit |
| 后端服务 | Supabase (PostgreSQL + Auth + Realtime + Storage) |
| 定位服务 | Core Location |

## 4. 功能详细描述

### 4.1 用户系统 [P0]

#### 4.1.1 注册/登录

**功能描述**：玩家通过Apple ID或邮箱注册登录游戏。

**用户故事**：
- 作为新玩家，我希望快速注册进入游戏
- 作为老玩家，我希望登录后恢复我的游戏数据

**功能规格**：

| 登录方式 | 说明 |
|----------|------|
| Sign in with Apple | 主要登录方式，一键授权 |
| 邮箱+密码 | 备选方式 |

**业务规则**：
- 新用户注册后需设置游戏昵称（2-12字符，不可重复）
- 昵称设置后30天内不可修改

#### 4.1.2 玩家资料

**功能描述**：展示玩家的基本信息和游戏数据。

**资料字段**：

| 字段 | 说明 |
|------|------|
| 头像 | 可上传自定义头像 |
| 昵称 | 游戏内显示名称 |
| 等级 | 根据经验值计算 |
| 领地数量 | 拥有的领地总数 |
| 领地总面积 | 所有领地面积之和 |
| 加入时间 | 注册日期 |

### 4.2 圈地系统 [P0]

#### 4.2.1 GPS路径记录

**功能描述**：玩家开启"圈地模式"后，系统持续记录GPS轨迹。

**用户故事**：
- 作为玩家，我希望开始行走时一键开启记录
- 作为玩家，我希望在地图上实时看到我走过的路径
- 作为玩家，我希望暂停记录去处理其他事情

**功能规格**：

| 功能点 | 说明 |
|--------|------|
| 开始记录 | 点击"开始圈地"按钮启动GPS追踪 |
| 实时轨迹 | 地图上用虚线显示当前行走路径 |
| 暂停/继续 | 支持暂停记录，暂停期间不记录轨迹 |
| 放弃 | 可随时放弃当前圈地，清除未闭合路径 |

**业务规则**：
- GPS采样频率：每5秒或移动10米记录一个点
- 最小路径点数：10个点
- 最大记录时长：4小时
- 后台运行：支持App进入后台时继续记录

**技术要点**：
- 使用Core Location的`startUpdatingLocation`
- 请求`Always`定位权限以支持后台定位
- 使用`allowsBackgroundLocationUpdates`

#### 4.2.2 闭环检测与领地生成

**功能描述**：当玩家路径形成闭环时，系统自动检测并生成领地。

**用户故事**：
- 作为玩家，我希望走回起点时自动完成圈地
- 作为玩家，我希望看到我圈下的领地范围

**闭环判定规则**：

| 条件 | 阈值 |
|------|------|
| 终点与起点距离 | ≤ 30米 |
| 最小围成面积 | ≥ 100平方米 |
| 最大围成面积 | ≤ 1平方公里（防作弊） |
| 路径自交检测 | 不允许路径自我交叉 |

**领地生成流程**：

```
开始圈地 → 记录路径 → 检测闭环 → 验证面积 → 检查重叠 → 生成领地 → 入库
                ↓
            未闭环则继续记录
```

**重叠处理规则**：
- 与自己已有领地重叠：合并为一个领地
- 与他人领地重叠：重叠部分归属于新圈地者（领地争夺）
- 完全被他人领地包含：圈地失败，提示"该区域已被占领"

#### 4.2.3 领地可视化展示

**功能描述**：在地图上以不同颜色展示各玩家的领地。

**视觉规格**：

| 元素 | 样式 |
|------|------|
| 自己的领地 | 绑绿色半透明填充，深绿色边框 |
| 他人的领地 | 红色半透明填充，深红色边框 |
| 正在圈的路径 | 黄色虚线 |
| 领地标签 | 显示领地名称和面积 |

**交互**：
- 点击领地：弹出详情卡片（所有者、面积、建筑列表）
- 长按自己的领地：进入领地管理（重命名、查看建筑、放弃领地）

### 4.3 探索系统 [P1]

#### 4.3.1 POI发现与展示

**功能描述**：基于玩家位置，在地图上展示附近可探索的POI。

**POI类型与图标**：

| POI类型 | 图标 | 产出物资类型 |
|---------|------|--------------|
| 医院 | 🏥 | 医疗用品 |
| 超市/商店 | 🏪 | 食物、日用品 |
| 加油站 | ⛽ | 燃料 |
| 工厂 | 🏭 | 建材、零件 |
| 学校 | 🏫 | 书籍、工具 |
| 警察局 | 🚔 | 武器、防具 |
| 住宅区 | 🏠 | 随机物资 |

**显示规则**：
- 显示范围：玩家周围500米内的POI
- 数据来源：MapKit的`MKLocalSearch`
- 刷新频率：玩家移动超过100米时刷新

#### 4.3.2 POI探索与奖励

**功能描述**：玩家到达POI附近可进行探索，获得物资奖励。

**探索条件**：
- 距离POI ≤ 50米
- 该POI今日未被该玩家探索过

**探索流程**：

```
到达POI范围 → 点击探索 → 播放探索动画(3秒) → 展示获得物资 → 物资入背包
```

**奖励机制**：

| POI类型 | 基础奖励 | 稀有奖励(10%概率) |
|---------|----------|-------------------|
| 医院 | 绷带x2, 止痛药x1 | 急救包x1 |
| 超市 | 罐头x2, 水x2 | 压缩饼干x1 |
| 加油站 | 汽油x1 | 柴油发电机燃料x1 |
| 工厂 | 钢材x2, 螺丝x5 | 电子元件x1 |
| 学校 | 纸张x3 | 技能书x1 |
| 警察局 | 警棍x1 | 防弹衣碎片x1 |

**冷却机制**：
- 单个POI：该玩家24小时后可再次探索
- 全局限制：每日最多探索20个POI

### 4.4 建造系统 [P1]

#### 4.4.1 建筑类型

**功能描述**：玩家可在自己的领地上建造各类功能建筑。

**建筑列表**：

| 建筑 | 功能 | 建造材料 | 建造时间 |
|------|------|----------|----------|
| 庇护所 | 设为重生点，每日恢复体力 | 木材x20, 钢材x10 | 1小时 |
| 仓库 | 增加背包容量+50 | 木材x30, 钢材x20 | 2小时 |
| 发电机 | 解锁高级建筑，为领地供电 | 钢材x30, 电子元件x5, 燃料x10 | 3小时 |
| 信号塔 | 扩大无线电通讯范围 | 钢材x40, 电子元件x10 | 4小时 |
| 防御塔 | 降低领地被夺取概率 | 钢材x50, 武器零件x5 | 5小时 |
| 交易站 | 可发布交易，吸引附近玩家 | 木材x40, 钢材x30 | 3小时 |

#### 4.4.2 建造规则

**前置条件**：
- 领地面积 ≥ 500平方米才能建造
- 每500平方米可建造1个建筑
- 部分建筑需要前置建筑（如高级建筑需要发电机）

**建造流程**：

```
选择领地 → 选择建筑类型 → 选择建造位置 → 确认消耗材料 → 开始建造 → 等待完成
```

**建筑升级**：
- 每个建筑可升级至3级
- 升级消耗材料为基础材料的1.5倍/2倍
- 升级效果提升50%/100%

### 4.5 背包系统 [P1]

#### 4.5.1 物资管理

**功能描述**：管理玩家获得的各类物资。

**背包规格**：

| 属性 | 默认值 | 说明 |
|------|--------|------|
| 基础容量 | 100格 | 每种物资占1格 |
| 堆叠上限 | 99个 | 同种物资最多堆叠99 |

**物资分类**：

| 分类 | 包含物资 |
|------|----------|
| 食物 | 罐头、水、压缩饼干等 |
| 医疗 | 绷带、止痛药、急救包等 |
| 材料 | 木材、钢材、电子元件等 |
| 燃料 | 汽油、柴油等 |
| 特殊 | 技能书、稀有道具等 |

**操作**：
- 查看详情：点击物资查看描述和用途
- 使用：消耗类物资可直接使用（如食物恢复体力）
- 丢弃：可丢弃不需要的物资

---

### 4.6 交易系统 [P2]

#### 4.6.1 玩家交易

**功能描述**：玩家之间可以交换物资。

**交易方式**：

| 方式 | 说明 |
|------|------|
| 面对面交易 | 双方距离≤100米，实时交换 |
| 交易站交易 | 在交易站发布求购/出售信息 |

**面对面交易流程**：

```
发起方选择交易对象 → 选择提供物资 → 发送交易请求
                            ↓
接收方收到请求 → 查看对方物资 → 选择交换物资 → 确认交易
                            ↓
                    双方确认 → 物资交换 → 交易完成
```

**交易站机制**：
- 发布交易信息需消耗少量物资作为手续费
- 交易信息对1公里内玩家可见
- 信息有效期24小时

#### 4.6.2 安全机制

- 交易需双方确认
- 交易记录保留30天
- 举报机制：可举报欺诈交易

---

### 4.7 通讯系统 [P2]

#### 4.7.1 无线电频道

**功能描述**：基于地理位置的公共聊天频道。

**频道类型**：

| 频道 | 范围 | 说明 |
|------|------|------|
| 本地频道 | 1公里 | 附近玩家公共聊天 |
| 领地频道 | 同一领地 | 在某领地内的玩家 |
| 全服频道 | 无限制 | 需消耗"广播器"道具 |

**功能规格**：
- 消息保留：24小时
- 发送频率：每分钟最多5条
- 内容审核：敏感词过滤

#### 4.7.2 私信系统

**功能描述**：玩家之间一对一通讯。

**功能规格**：
- 需要双方互相添加好友
- 或有过交易记录
- 消息保留：7天
- 支持发送物资截图

### 6.2 核心模块说明

#### 6.2.1 定位模块 (LocationManager)

**职责**：
- 管理GPS定位权限请求
- 持续追踪用户位置
- 支持后台定位
- 路径点采集与去噪

**关键技术**：
- `CLLocationManager`
- 后台定位模式：`Background Modes > Location updates`
- 显著位置变化：`startMonitoringSignificantLocationChanges()`

#### 6.2.2 地图模块 (MapKit Integration)

**职责**：
- 渲染地图底图
- 绘制领地多边形
- 展示POI标记
- 处理地图交互

**关键技术**：
- `MKMapView` / `Map` (SwiftUI)
- `MKPolygon` 领地绘制
- `MKAnnotation` POI标记
- `MKLocalSearch` POI搜索

#### 6.2.3 数据同步模块 (SyncManager)

**职责**：
- 管理与Supabase的数据同步
- 处理离线数据缓存
- 冲突解决策略

**同步策略**：
- 领地数据：实时同步
- 背包数据：5秒节流同步
- 消息数据：WebSocket实时推送

#### 6.2.4 Supabase服务

**Auth服务**：
- Sign in with Apple 集成
- 邮箱密码认证
- Session管理

**Database服务**：
- PostGIS扩展处理地理数据
- RLS策略保护数据安全
- 复杂查询（领地重叠检测等）

**Realtime服务**：
- 订阅领地变更
- 实时聊天消息
- 交易请求通知

**Edge Functions**：
- 领地重叠计算
- 奖励发放逻辑
- 防作弊检测

### 6.3 数据流示例

**圈地完成数据流**：

```
用户点击"结束圈地"
       │
       ▼
LocationManager 停止GPS追踪
       │
       ▼
GameManager 检测闭环条件
       │
       ├── 失败 → 提示用户"未形成闭环"
       │
       ▼ 成功
计算多边形面积
       │
       ├── 面积不合规 → 提示用户
       │
       ▼ 合规
调用 Supabase Edge Function 检测重叠
       │
       ├── 完全被包含 → 提示用户"该区域已被占领"
       │
       ▼ 可创建
Supabase Database 创建领地记录
       │
       ▼
Supabase Realtime 广播领地变更
       │
       ▼
其他在线用户收到通知，地图刷新
```
